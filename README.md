# Тестовое задание компании "Моторика":<br>распознавание жестов

В [**ноутбуке**](https://nbviewer.org/github/sidorov-works/motorica/blob/main/Motorica.ipynb) представлено решение тестового задания для приема на стажировку в компанию ["Моторика"](https://motorica.org/home).

## Задача
Построить модель распознавания жестов на основании временных рядов показаний датчиков, закрепленных на руку *пилота* (*пилот*
 – человек, с которого собираются данные).

Пилоту поступают команды о выполнении жестов, пилот после получения команды с некоторой естественной задержкой выполняет их.

**Целевая переменная** представляет из себя закодированную команду, поданную пользователю на выполнение жеста в каждый момент времени.

## Данные

Данные представляют из себя временные ряды, где в каждый момент времени фиксируются показания от датчиков.

В **тестовой выборке** требуется разметить каждое измерение кодом выполняемого жеста (задержка уже не учитывается)

## Метрика оценки качества решения

Качество решения оценивается по метрике *Mean F-score* на отложенной тестовой выборке (в формате [соревнования](https://www.kaggle.com/competitions/motorica-skillfactory-intership-test-task-2024-10/overview) *Kaggle*).

## Что сделано

1. Проведен анализ предоставленных данных, выполнено преобразование массивов в двумерный вид, пригодный для дальнейшей подготовки.
2. Разработан алгоритм разделения временных рядов показаний датчиков на фактические наблюдения (жесты). В основе алгоритма лежит поиск локальных максимумов суммарного абсолютного градиента последовательностей показаний датчиков (такие "экстремумы" служат основой для разграничения фактически выполняемых жестов).
3. Выполнена разметка данных по фактическим наблюдениям.
4. В качестве *baseline* реализовано решение задачи на базе логистической регрессии, в качестве значений признаков использовались медианы показаний датчиков в границах размеченных фактических наблюдений.
5. Осуществлен подбор оптимальных гиперпараметров.
6. Выполнен *submit* полученных результатов на *Kaggle*.

## Результаты работы

В открытом [лидерборде](https://www.kaggle.com/competitions/motorica-skillfactory-intership-test-task-2024-10/leaderboard) соревнования представленное решение оценивается значением метрики 0.98.

> Полученный результат можно считать хорошим, поскольку изначальные условия не предполагает 100-процентного попадания: нам не известен принцип (или алгоритм), по которому атворы задания разметили отложенную выборку. И даже в случае абсолютно точного предсказания последовательности жестов неизбежно будут возникать некоторые расхождения в разметке отдельных точек (измерений) на границах жестов.

## Развитие проекта

Дальнейшее развитие проекта может идти в направлении, когда нам не известно количество временных шагов от одной команды до другой (да и сами команды могут отсуствовать на отложенной тестовой выборке). В этом случае обученная модель должна последовательно обрабатывать поступающие отдельные измерения (как в режиме реального времени), определять переход к следующему жесту "на лету" и предсказывать его класс.
